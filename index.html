HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Semantic HTML elements: the name of an element describes content and function in a webpage and provides clarity for the greatest number of viewers
  • Heading tags: allow information in a page to be organized and accessible for screen readers

CSS

  • Inline style: style is directly applied to HTML file, corresponds to specific tags and individual elemnts
  • Internal style:use of a style attribute embedded in HTML header
  • External style:a separate file containing CSS rules linked to HTML header
  • selectors: defines the element or attributes that declarations and values will be applied to

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates new branch and switches to it
  • feature/add-html: staging area for changed files to be integrated from the working directory to Git repository
  • git add -A: stages changes made to files in preparation for commit from working directory to Git repository
  • git commit -m: initiates commit to the repository with notes pertaining to content changes

JavaScript

  • Variables: keyword let declares a variable with its assigned value
  • Strings: the use of single or double quotation marks around a sequence of code
  • Dynamic variables: have no direct association with value type
  • Array: a structure that provides storage for multiple values in a single reference
  • Script loading: JavaScript is recommended to be linked to the bottom of an HTML doc so its effects are seen after HTML and CSS content loads
